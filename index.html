Proyecto final módulo 1: Glosario

Introducción a la programación

En este glosario se presentan los conceptos clave del primer módulo de desarrollo front-end, que cubre los fundamentos de HTML y CSS, así como herramientas y prácticas esenciales para la creación de sitios web. Los términos incluyen atributos HTML, propiedades CSS, el modelo de caja, el manejo de colores y bordes, y conceptos relacionados con el control de versiones (Git) y herramientas de desarrollo (DevTools). Este glosario está diseñado para ser una referencia práctica y accesible para estudiantes que están comenzando en el desarrollo web.

A

Atributos HTML:

Propiedades adicionales que se agregan a las etiquetas HTML para proporcionar información o configurar su comportamiento. Ejemplos: class, id, src, alt.

B

Border:

Define el borde alrededor de un elemento HTML. Ejemplo: solid (continuo), dashed (discontinuo), dotted (de puntos).

Border-radius:

Controla las esquinas redondeadas de un elemento. Ejemplo: border-radius: 10px.

Border-color:

Define el color del borde, utilizando formatos como hexadecimal, RGB, etc.

C

Chromium Project:

Proyecto de navegador web de código abierto iniciado por Google. Es la base para navegadores como Google Chrome, Microsoft Edge, Opera, Brave y Vivaldi. Sus DevTools funcionan de forma similar en todos ellos.

Clear:

Comando de la terminal que limpia la pantalla actual.

Colores en CSS:

Métodos para definir colores:

  • Nombre: Ejemplo: red, blue.
  • Hexadecimal: Ejemplo: #FF5733.
  • RGB/RGBA: Ejemplo: rgb(255, 87, 51).
  • HSL/HSLA: Ejemplo: hsl(12, 100%, 50%).
  • Otros formatos avanzados: Ejemplo: HWB, LCH, OKLCH.

Comandos básicos de Git:

Instrucciones esenciales para interactuar con Git:

  • git clone <url_repo>: Clona un repositorio remoto en tu máquina local.
  • git status: Muestra el estado actual del repositorio.
  • git add <nombre_archivo>: Agrega un archivo al área de preparación.
  • git commit -m "comentario": Guarda cambios con un mensaje descriptivo.
  • git diff <nombre_archivo>: Muestra diferencias entre versiones.
  • git push: Sube cambios al repositorio remoto.
  • git pull: Descarga y combina cambios desde el repositorio remoto.

Comandos básicos de la terminal:

Operaciones comunes en la terminal:

  • ls: Muestra el contenido del directorio actual.
  • cd <directorio>: Cambia al directorio especificado.
  • pwd: Muestra la ruta del directorio actual.
  • mkdir <directorio>: Crea un nuevo directorio.
  • touch <nombre_archivo>: Crea un archivo vacío.
  • cp <archivo>: Copia un archivo a otra ubicación.
  • mv <dir_actual> <dir_nuevo>: Mueve o renombra archivos/directorios.
  • rm <archivo>: Elimina un archivo.
  • code .: Abre el directorio actual en Visual Studio Code.

Conventional Commits:

Convención para redactar mensajes de commit claros mediante prefijos:

  • chore: Cambios menores o de mantenimiento.
  • docs: Cambios en la documentación.
  • feat: Nueva funcionalidad.
  • fix: Corrección de errores.
  • style: Cambios de formato o estilo sin impacto funcional.

CSS (Cascading Style Sheets):

Lenguaje que define cómo se presentan los elementos HTML en la pantalla o en otros dispositivos.

D

DevTools (Herramientas de desarrollo):

Conjunto de herramientas incluidas en los navegadores para depurar, analizar y mejorar el rendimiento de sitios web. Ejemplo: Chrome DevTools.

Discord:

Plataforma de comunicación con chat de texto, voz y video, útil para equipos de desarrollo y comunidades técnicas.

Display:

Propiedad CSS que define el comportamiento de un elemento:

  • Block Boxes: Ocupan todo el ancho disponible y comienzan en una nueva línea.
  • Inline Boxes: Ocupan solo el espacio necesario y permanecen en la misma línea.

E

Especificidad en CSS:

Prioridad para aplicar reglas CSS. Si varias reglas tienen el mismo nivel, se aplica la que esté más abajo en el archivo CSS.

Estructura de Archivos en Proyectos Web:

Organización de archivos y carpetas en un proyecto para mejorar la claridad y facilitar la colaboración. Ejemplo:

  • HTML: Archivos base (index.html).
  • CSS: Estilos (styles.css).
  • JS: Scripts (app.js).
  • Assets: Recursos como imágenes y fuentes.

F

Font-family:

Propiedad que define la fuente del texto. Puede utilizar fuentes estándar (Web Safe) o fuentes personalizadas, como las de Google Fonts.

Font-size:

Controla el tamaño del texto, definido en píxeles, em, rem o porcentajes.

Font-style:

Estilo del texto, como normal, italic o oblique.

Font-weight:

Grosor del texto. Valores: normal, bold, o un rango entre 100 y 900.

G

Git:

Sistema de control de versiones que permite rastrear cambios en el código, mantener un historial y colaborar en proyectos.

Github:

Plataforma para alojar y gestionar proyectos utilizando Git, ideal para desarrollo colaborativo.

Google Drive:

Plataforma de almacenamiento en la nube para guardar, compartir y colaborar en documentos.

H

Herencia en CSS:

Propiedades CSS, como color o font-family, que los elementos hijo heredan de sus elementos padre.

I

IDE (Integrated Development Environment):

Herramienta que combina funciones para escribir, probar y depurar código.

Inspector Web:

Herramienta en navegadores para ver, editar y depurar HTML/CSS en tiempo real. Acceso: clic derecho en un elemento y seleccionar "Inspeccionar".

IA (Inteligencia Artificial):

Tecnología que permite a las máquinas realizar tareas que requieren inteligencia humana, como automatización, generación de ideas y resolución de problemas.

L

LLMs (Large Language Models):

Modelos de lenguaje que generan contenido o completan tareas. Su efectividad depende del diseño de prompts adecuados.

M

Modelo de Caja (Box Model):

Concepto clave en diseño web que estructura los elementos como una caja compuesta de:

  • Content: Área donde se muestra el contenido.
  • Padding: Espacio entre el contenido de un elemento y su borde.
  • Border: Contorno del elemento.
  • Margin: Espacio exterior que separa un elemento de otros elementos en la página.

N

Node.js:

Entorno de ejecución para JavaScript que permite ejecutar código del lado del servidor. Se utiliza para construir aplicaciones web escalables y rápidas.

P

Position:

Propiedad CSS que controla cómo se posiciona un elemento. Tipos:

  • Static: Posición por defecto.
  • Relative: Permite mover el elemento respecto a su posición original.
  • Absolute: Posición respecto al contenedor padre.
  • Fixed: Posición fija en la ventana.
  • Sticky: Se mantiene "pegado" en un punto definido al hacer scroll.

Prompt:

nstrucción dada a un modelo de IA para generar una respuesta o realizar una tarea.

R

Ramas (Branches):

Versiones paralelas de los archivos en un repositorio. La rama inicial por defecto se llama main.

Remotes:

Conexión entre el repositorio local y uno remoto alojado en un servidor (por ejemplo, en GitHub). Remote por defecto: origin.

S

Selectores CSS:

Parte de una regla CSS que selecciona elementos HTML para aplicar estilos. Tipos:

  • Etiqueta: Selecciona elementos por nombre (ejemplo: <h1>).
  • Clase: Selecciona elementos con el atributo class (ejemplo: .mi-clase).
  • Id: Selecciona un elemento único con el atributo id (ejemplo: #mi-id).

Sintaxis CSS:

Consiste en selector, propiedad y valor.

Sintaxis HTML:

Uso de etiquetas con una estructura que incluye:

  • Opening Tag: Etiqueta de apertura.
  • Content: Contenido del elemento.
  • Closing Tag: Etiqueta de cierre.

T

Trello:

Herramienta de gestión de proyectos basada en tableros, listas y tarjetas.

V

Visual Studio Code:

IDE ligero, versátil y de código abierto, con soporte para extensiones y múltiples lenguajes.
